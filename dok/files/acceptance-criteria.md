# Acceptance Criteria — Инвестиционное золото

> Формат: Given / When / Then (BDD)

---

## 1. Каталог и поиск

### US-01 — Каталог на главной

**AC-01.1**
- **Given** пользователь открывает главную страницу
- **When** страница загружается
- **Then** отображается сетка товаров (минимум 8 карточек), каждая с фото, названием, весом и ценой

**AC-01.2**
- **Given** пользователь не авторизован
- **When** страница загружается
- **Then** каталог доступен без регистрации, кнопка "Купить" активна

**AC-01.3**
- **Given** цены обновились на бирже
- **When** страница перезагружается
- **Then** цены на карточках отражают актуальные котировки

---

### US-02 — Фильтрация каталога

**AC-02.1**
- **Given** пользователь находится в каталоге
- **When** выбирает фильтр по типу "Монеты"
- **Then** отображаются только монеты, остальные товары скрыты

**AC-02.2**
- **Given** пользователь применил фильтр по весу "10г"
- **When** каталог обновляется
- **Then** отображаются только товары весом 10г, количество результатов показано

**AC-02.3**
- **Given** пользователь применил несколько фильтров
- **When** нажимает "Сбросить фильтры"
- **Then** все фильтры снимаются, каталог показывает все товары

---

### US-03 — Карточка товара

**AC-03.1**
- **Given** пользователь смотрит на карточку товара
- **When** карточка отрисована
- **Then** видны: фото, название, вес в граммах, цена покупки (RSD), цена продажи (RSD), наличие на складе

**AC-03.2**
- **Given** товар отсутствует на складе
- **When** карточка отрисована
- **Then** отображается бейдж "Нет в наличии", кнопка "Купить" заменена на "Уведомить"

---

### US-04 — Страница товара

**AC-04.1**
- **Given** пользователь кликает на товар
- **When** открывается страница
- **Then** отображается: галерея фото (минимум 2), описание, характеристики (вес, проба, производитель), кнопка "Добавить в корзину"

**AC-04.2**
- **Given** пользователь на странице товара
- **When** прокручивает вниз
- **Then** видна секция "Похожие товары" (минимум 4 позиции)

---

### US-05 — Поиск

**AC-05.1**
- **Given** пользователь вводит "10 грамм" в строку поиска
- **When** нажимает Enter или иконку поиска
- **Then** результаты показаны в течение 500мс, релевантные товары отображены первыми

**AC-05.2**
- **Given** поиск не дал результатов
- **When** страница результатов открыта
- **Then** показано сообщение "Ничего не найдено" и предложены популярные товары

---

## 2. Живые котировки

### US-06 — Текущая цена золота

**AC-06.1**
- **Given** пользователь открывает любую страницу
- **When** страница загружается
- **Then** в шапке видна текущая цена XAU в RSD, EUR, USD с временем последнего обновления

**AC-06.2**
- **Given** цена изменилась
- **When** прошло не более 60 секунд
- **Then** цена в шапке обновляется автоматически без перезагрузки страницы

**AC-06.3**
- **Given** API котировок недоступен
- **When** страница загружается
- **Then** показана последняя известная цена с пометкой "Данные могут быть устаревшими"

---

### US-07 — График цен

**AC-07.1**
- **Given** пользователь открывает страницу котировок
- **When** страница загружается
- **Then** отображается интерактивный график с периодами: 1Д / 1Н / 1М / 1Г

**AC-07.2**
- **Given** пользователь наводит курсор на точку графика
- **When** курсор на графике
- **Then** показывается тултип с датой, временем и ценой

---

### US-08 — Автообновление цен на товарах

**AC-08.1**
- **Given** цена золота изменилась более чем на 0.5%
- **When** пользователь находится на странице каталога
- **Then** цены пересчитываются автоматически, показывается анимация обновления

---

## 3. Корзина и Wishlist

### US-09 — Корзина без регистрации

**AC-09.1**
- **Given** гость добавляет товар в корзину
- **When** нажимает "Добавить в корзину"
- **Then** счётчик корзины в шапке увеличивается, появляется уведомление о добавлении

**AC-09.2**
- **Given** гость добавил товары в корзину и закрыл браузер
- **When** возвращается на сайт в течение 24 часов
- **Then** корзина сохранена (localStorage)

---

### US-10 — Wishlist

**AC-10.1**
- **Given** пользователь нажимает иконку сердца на карточке товара
- **When** действие выполнено
- **Then** товар добавлен в список желаний, иконка стала активной (закрашенной)

**AC-10.2**
- **Given** товар уже в wishlist
- **When** пользователь нажимает иконку снова
- **Then** товар удаляется из wishlist, иконка возвращается в неактивное состояние

---

### US-12 — Количество в корзине

**AC-12.1**
- **Given** пользователь в корзине изменяет количество товара
- **When** вводит новое значение
- **Then** итоговая сумма пересчитывается мгновенно

**AC-12.2**
- **Given** пользователь пытается ввести количество "0" или отрицательное
- **When** значение введено
- **Then** значение принудительно устанавливается в "1", ошибка не выбрасывается

---

### US-13 — Изменение цены в корзине

**AC-13.1**
- **Given** цена товара изменилась после добавления в корзину
- **When** пользователь открывает корзину
- **Then** отображается предупреждение "Цена изменилась с [старая] до [новая] RSD"

---

## 4. Регистрация и аккаунт

### US-14 — Регистрация

**AC-14.1**
- **Given** пользователь заполняет форму регистрации
- **When** вводит валидный email и пароль (минимум 8 символов)
- **Then** аккаунт создан, отправлено письмо подтверждения, пользователь перенаправлен в личный кабинет

**AC-14.2**
- **Given** email уже зарегистрирован
- **When** пользователь пытается зарегистрироваться
- **Then** показана ошибка "Email уже используется", предложен вход или восстановление пароля

**AC-14.3**
- **Given** пользователь вводит невалидный email (без @)
- **When** нажимает "Зарегистрироваться"
- **Then** форма не отправляется, поле подсвечено красным с подсказкой

---

### US-16 — История заказов

**AC-16.1**
- **Given** авторизованный пользователь открывает "Мои заказы"
- **When** страница загружается
- **Then** показан список заказов: номер, дата, состав, статус, сумма

**AC-16.2**
- **Given** пользователь кликает на заказ
- **When** открывается детальная страница
- **Then** видны: состав заказа, адрес доставки, способ оплаты, трек-номер (если есть)

---

## 5. Оформление заказа

### US-19 — Способ получения

**AC-19.1**
- **Given** пользователь на шаге оформления
- **When** выбирает "Доставка"
- **Then** появляется форма адреса с полями: улица, город, индекс, имя получателя

**AC-19.2**
- **Given** пользователь выбирает "Самовывоз"
- **When** опция выбрана
- **Then** показан список адресов офисов с картой, форма адреса скрыта

---

### US-20 — Оплата

**AC-20.1**
- **Given** пользователь на шаге оплаты
- **When** выбирает "Банковская карта"
- **Then** открывается защищённая форма ввода карты (PCI DSS), в адресной строке HTTPS

**AC-20.2**
- **Given** оплата прошла успешно
- **When** транзакция подтверждена
- **Then** пользователь перенаправлен на страницу "Спасибо за заказ" с номером заказа

**AC-20.3**
- **Given** оплата отклонена
- **When** банк вернул ошибку
- **Then** показано сообщение об ошибке, корзина сохранена, предложено попробовать другую карту

---

### US-21 — Email подтверждения

**AC-21.1**
- **Given** заказ успешно оформлен
- **When** в течение 2 минут
- **Then** покупатель получает email с: номером заказа, составом, суммой, сроками и контактами поддержки

---

## 6. Контент

### US-26 — Блог

**AC-26.1**
- **Given** пользователь открывает блог
- **When** страница загружается
- **Then** статьи отображаются в виде сетки с: обложкой, заголовком, датой, категорией и кратким описанием

**AC-26.2**
- **Given** пользователь читает статью
- **When** прокручивает вниз
- **Then** в конце показаны 3 похожие статьи

---

## 7. Контакты

### US-29 — Контактная форма

**AC-29.1**
- **Given** пользователь заполняет форму обратной связи
- **When** нажимает "Отправить" с валидными данными
- **Then** форма отправлена, показано подтверждение, менеджер получил email с данными

**AC-29.2**
- **Given** пользователь не заполнил обязательные поля
- **When** нажимает "Отправить"
- **Then** форма не отправляется, незаполненные поля подсвечены с подсказками

---

## 8. Администратор

### US-32 — Управление товарами

**AC-32.1**
- **Given** администратор в панели управления
- **When** создаёт новый товар
- **Then** товар появляется в каталоге после сохранения (не более 5 секунд)

**AC-32.2**
- **Given** администратор удаляет товар
- **When** подтверждает удаление
- **Then** товар исчезает из каталога, заказы с этим товаром сохраняются в истории

---

### US-33 — Управление заказами

**AC-33.1**
- **Given** администратор открывает список заказов
- **When** применяет фильтр по статусу
- **Then** показаны только заказы с выбранным статусом

**AC-33.2**
- **Given** администратор меняет статус заказа
- **When** сохраняет изменение
- **Then** покупатель получает email-уведомление о смене статуса

---

### US-35 — Уведомления администратора

**AC-35.1**
- **Given** новый заказ оформлен
- **When** в течение 1 минуты
- **Then** администратор получает email с деталями заказа

---

## Нефункциональные требования

| Требование | Критерий |
|---|---|
| Производительность | LCP < 2.5с, FID < 100мс (Core Web Vitals) |
| Доступность | WCAG 2.1 AA |
| Безопасность | HTTPS, CSP headers, защита от XSS/CSRF |
| Мобильная версия | Корректное отображение от 320px |
| SEO | SSR/SSG на Next.js, мета-теги на всех страницах |
| Uptime | 99.5% в рабочее время |
