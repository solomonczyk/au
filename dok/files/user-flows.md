# User Flows — Интернет-магазин инвестиционного золота

---

## Flow 1: Покупка товара (Happy Path)

```
Главная страница
    └─► Каталог (фильтры по типу/весу)
        └─► Страница товара (галерея, цена, описание)
            └─► Добавить в корзину
                └─► [Авторизован?]
                    ├─ НЕТ ─► Вход / Регистрация ──────┐
                    └─ ДА ──────────────────────────────┘
                                    ▼
                        Шаг 1: Контактные данные
                                    ▼
                        Шаг 2: Способ доставки
                        (Курьер / Самовывоз)
                                    ▼
                        Шаг 3: Оплата
                        (Карта / Перевод / Наличные)
                                    ▼
                        [Оплата прошла?]
                         ├─ НЕТ ─► Ошибка → повторить
                         └─ ДА  ─► Страница "Спасибо"
                                        ▼
                                Email с деталями заказа
```

**Тригеры нотификаций:**
- Покупатель: email при создании заказа
- Администратор: email при новом заказе
- Покупатель: email при смене статуса заказа

---

## Flow 2: Регистрация пользователя

```
Кнопка "Войти / Зарегистрироваться"
    └─► Форма регистрации
            └─► [email уникален?]
                ├─ НЕТ ─► Сообщение "Email уже используется"
                │              └─► Предложить восстановление пароля
                └─ ДА  ─► Аккаунт создан в БД
                                └─► Письмо с подтверждением email
                                        └─► [Пользователь перешёл по ссылке?]
                                            ├─ НЕТ ─► Повторная отправка
                                            └─ ДА  ─► Email подтверждён
                                                            └─► Личный кабинет
```

**Правила валидации формы:**
- Email: формат, уникальность
- Пароль: минимум 8 символов, 1 цифра
- Телефон: опционально, формат +381...

## Flow 3: Обработка заказа (Администратор)

```
Новый заказ оформлен
    └─► Уведомление на email администратора
        └─► /admin/orders — заказ со статусом "Новый"
                └─► Открывает детали заказа
                        └─► [Оплата подтверждена банком?]
                            ├─ НЕТ ─► Связаться с покупателем
                            │             └─► Отмена заказа (если нет ответа)
                            └─ ДА  ─► Статус: "Оплачен"
                                            └─► Подготовка товара к отгрузке
                                                └─► Статус: "Подготовлен"
                                                        └─► Передача в доставку
                                                                └─► Вводит трек-номер
                                                                └─► Статус: "Отправлен"
                                                                └─► Email покупателю с трекингом
                                                                        └─► Товар доставлен
                                                                                └─► Статус: "Завершён"
```

---

## Flow 4: Поиск товара

```
Пользователь вводит запрос в поиск
    └─► [Автодополнение — debounce 300мс]
            └─► [Нашлись результаты?]
                ├─ ДА  ─► Выбирает из подсказок → Страница товара
                └─► Нажимает Enter → Страница результатов поиска
                            └─► [Есть результаты?]
                                ├─ ДА  ─► Список товаров с фильтрами
                                └─ НЕТ ─► "Ничего не найдено"
                                              └─► Популярные товары
```

---

## Flow 5: Обновление цен (Система)

```
[Каждые 60 секунд]
    └─► API запрос к nFusion Solutions
            └─► [API доступен?]
                ├─ НЕТ ─► Последнее известное значение + пометка "устаревшие данные"
                └─ ДА  ─► Новые котировки получены
                                └─► Пересчёт цен товаров
                                        (спотовая цена × вес × наценка%)
                                └─► Обновление тикера в шапке
                                └─► [Цена изменилась > 0.5%?]
                                    └─ ДА ─► Обновление цен на карточках
                                                └─► [Товар в корзине?]
                                                    └─ ДА ─► Предупреждение пользователю
```

---

## Таблица статусов заказа

| Статус | Кто меняет | Нотификация |
|---|---|---|
| Новый | Система (автоматически) | Email администратору |
| Оплачен | Система / Администратор | — |
| Подготовлен | Администратор | — |
| Отправлен | Администратор | Email покупателю |
| Доставлен | Администратор / Покупатель | — |
| Завершён | Администратор | — |
| Отменён | Администратор / Система | Email покупателю |

---

## Критичные проверки (Guard Conditions)

| Условие | Что происходит |
|---|---|
| Товар закончился в процессе чекаута | Предупреждение, предложение уведомить |
| Цена изменилась в корзине | Уведомление, пересчёт |
| Сессия истекла при оформлении | Перенаправление на вход, корзина сохраняется |
| Платёж отклонён | Корзина сохраняется, предложить другой способ |
| Email не подтверждён | Возможность просматривать, но не оформлять заказ |
